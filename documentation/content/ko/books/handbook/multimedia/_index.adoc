[[multimedia-synopsis]]
== 요약

멀티미디어 장에서는 FreeBSD의 멀티미디어 지원에 대한 개요를 제공합니다. 멀티미디어 애플리케이션과 기술은 현대 컴퓨팅의 필수적인 부분이 되었으며, FreeBSD는 다양한 멀티미디어 하드웨어와 소프트웨어를 강력하고 안정적으로 지원합니다. 이 장에서는 오디오, 비디오, 이미지 처리와 같은 다양한 멀티미디어 구성 요소를 다룹니다. 또한 다양한 미디어 포맷과 코덱, 멀티미디어 제작 및 재생을 위한 도구와 애플리케이션에 대해서도 설명합니다. 또한 이 장에서는 멀티미디어 시스템 구성, 문제 해결 및 최적화에 대해서도 다룹니다. 멀티미디어 애호가든 전문 콘텐츠 제작자이든, FreeBSD는 멀티미디어 작업을 위한 강력한 플랫폼을 제공합니다. 이 장에서는 FreeBSD의 멀티미디어 기능을 최대한 활용하는 데 도움이 되는 유용한 정보와 실용적인 예제를 제공하여 시작하는 데 도움을 줍니다.

[[sound-setup]]
== 사운드 카드 설정하기

기본적으로 FreeBSD는 시스템에서 사용하는 사운드 카드를 자동으로 감지합니다. FreeBSD는 다양한 사운드 카드를 지원합니다. 지원되는 사운드 카드 목록은 man:sound[4]에서 확인할 수 있습니다.

[NOTE]
====
사운드 카드 모듈을 올바르게 감지하지 못한 경우에만 사운드 카드 모듈을 로드하면 됩니다.
====

시스템에 어떤 사운드 카드가 있는지 또는 어떤 모듈을 사용할지 알 수 없는 경우 다음 명령을 실행하여 `snd_driver` 메타드라이버를 로드할 수 있습니다:

[source, shell]
....
# kldload snd_driver
....

또는 부팅 시 드라이버를 모듈로 로드하려면 [.filename]#/boot/loader.conf# 에 다음 줄을 넣습니다:

[.programlisting]
....
snd_driver_load="YES"
....

[[sound-testing]]
=== 사운드 테스트하기

사운드 카드가 감지되었는지 확인하려면 다음 명령을 실행합니다:

[source, shell]
....
% dmesg | grep pcm
....

출력은 다음과 비슷해야 합니다:

[.programlisting]
....
pcm0: <Conexant CX20561 (Hermosa) (Analog 2.0+HP/2.0)> at nid 26,22 and 24 on hdaa0
pcm1: <Conexant CX20561 (Hermosa) (Internal Analog Mic)> at nid 29 on hdaa0
....

다음 명령을 사용하여 사운드 카드의 상태도 확인할 수 있습니다:

[source, shell]
....
# cat /dev/sndstat
....

출력은 다음과 비슷해야 합니다:

[.programlisting]
....
Installed devices:
pcm0: <Conexant CX20561 (Hermosa) (Analog 2.0+HP/2.0)> (play/rec) default
pcm1: <Conexant CX20561 (Hermosa) (Internal Analog Mic)> (rec)
....

만약 `pcm` 장치가 나열되지 않으면, 올바른 장치 드라이버가 로드되었는지 다시 한 번 확인합니다. 모든 것이 잘 되었다면, 이제 사운드 카드가 FreeBSD에서 작동할 것입니다.

man:beep[1]을 사용하여 약간의 소음을 발생시켜 사운드 카드가 작동 중임을 확인할 수 있습니다:

[source, shell]
....
% beep
....

[[graphics-card-sound]]
=== 그래픽 카드 사운드

그래픽 카드에는 자체 통합 사운드 장치가 함께 제공되는 경우가 많으며, 어떤 장치가 기본 장치로 사용되고 있는지 불분명할 수 있습니다. 확인하려면 dmesg를 실행하고 pcm 항목을 찾아 시스템에서 출력을 열거하는 방식을 확인합니다. 다음 명령을 실행합니다:

[source, shell]
....
% dmesg | grep pcm
....

출력은 다음과 같이 표시됩니다:

[.programlisting]
....
pcm0: <HDA NVidia (Unknown) PCM #0 DisplayPort> at cad 0 nid 1 on hdac0
pcm1: <HDA NVidia (Unknown) PCM #0 DisplayPort> at cad 1 nid 1 on hdac0
pcm2: <HDA NVidia (Unknown) PCM #0 DisplayPort> at cad 2 nid 1 on hdac0
pcm3: <HDA NVidia (Unknown) PCM #0 DisplayPort> at cad 3 nid 1 on hdac0
hdac1: HDA Codec #2: Realtek ALC889
pcm4: <HDA Realtek ALC889 PCM #0 Analog> at cad 2 nid 1 on hdac1
pcm5: <HDA Realtek ALC889 PCM #1 Analog> at cad 2 nid 1 on hdac1
pcm6: <HDA Realtek ALC889 PCM #2 Digital> at cad 2 nid 1 on hdac1
pcm7: <HDA Realtek ALC889 PCM #3 Digital> at cad 2 nid 1 on hdac1
....

그래픽 카드(NVIDIA(R))가 사운드 카드(Realtek(R)) 앞에 열거되어 있으며 사운드 카드가 `pcm4` 로 표시됩니다. 다음 명령을 실행하여 사운드 카드를 기본 장치로 사용하도록 시스템을 구성할 수 있습니다:

[source, shell]
....
# sysctl hw.snd.default_unit=4
....

이 변경 사항을 영구적으로 적용하려면 [.filename]#/etc/sysctl.conf# 에 다음 줄을 추가합니다:

[.programlisting]
....
hw.snd.default_unit=4
....

[[automatically-switching-headphones]]
=== 헤드폰으로 자동 전환하기

일부 시스템에서는 오디오 출력을 전환하는 데 어려움을 겪을 수 있지만, 다행히도 FreeBSD에서는 [.filename]#device.hints# 에서 자동 전환을 구성할 수 있습니다.

다음 명령을 실행하여 시스템이 오디오 출력을 열거하는 방식을 확인합니다:

[source, shell]
....
% dmesg | grep pcm
....

출력은 다음과 같이 표시됩니다:

[.programlisting]
....
pcm0: <Realtek ALC892 Analog> at nid 23 and 26 on hdaa0
pcm1: <Realtek ALC892 Right Analog Headphones> at nid 22 on hdaa0
....

[.filename]#/boot/device.hints# 에 다음 줄을 추가합니다:

[.programlisting]
....
hint.hdac.0.cad0.nid22.config="as=1 seq=15 device=Headphones" 
hint.hdac.0.cad0.nid26.config="as=2 seq=0 device=speakers"
....

[NOTE]
====
이 값은 위에 표시된 예시를 위한 값입니다. 시스템에 따라 다를 수 있습니다.
====

[[troubleshooting]]
=== 사운드 문제 해결

몇 가지 일반적인 오류 메시지와 해결 방법이 있습니다:

[[multimedia-sound-common-error-messages]]

[cols="1,1", frame="none", options="header"]
|===
| Error
| Solution

|`xxx: can't open /dev/dsp!`
|Type `fstat \| grep dsp` to check if another application is holding the device open. Noteworthy troublemakers are esound and KDE's sound support.

package:audio/pulseaudio[]를 사용하는 프로그램은 `hw.snd.default_unit` 의 변경 사항을 적용하기 위해 package:audio/pulseaudio[] 데몬을 다시 시작해야 할 수 있습니다. 또는 package:audio/pulseaudio[] 설정을 즉석에서 변경할 수도 있습니다. man:pacmd[1]은 package:audio/pulseaudio[] 데몬에 대한 명령줄 연결을 엽니다:

[source, shell]
....
# pacmd
Welcome to PulseAudio 14.2! Use "help" for usage information.
>>>
....

다음 명령은 이전 예제에서와 같이 기본 싱크를 카드 번호 4로 변경합니다:

[.programlisting]
....
set-default-sink 4
....

[WARNING]
====
명령줄 인터페이스를 종료할 때 `exit` 명령을 사용하지 마세요. package:audio/pulseaudio[] 데몬이 종료됩니다. 대신 kbd:[Ctrl+D]를 사용하세요.
====

=== Audacious

Audacious는 오픈 소스 오디오 플레이어입니다. XMMS의 후손인 이 플레이어는 다른 작업에서 컴퓨터 리소스를 빼앗지 않고 원하는 방식으로 음악을 재생합니다.

[[moc-music-on-console]]
=== MOC (music on console)

MOC (music on console는 강력하고 사용하기 쉽도록 설계된 콘솔 오디오 플레이어입니다.

MOC는 출력 버퍼를 별도의 스레드에서 처리하기 때문에 시스템이나 I/O 로드에 관계없이 원활하게 재생됩니다. 현재 파일을 재생하는 동안 다음 재생할 파일이 미리 캐시되므로 파일 간에 끊김이 발생하지 않습니다.

MOC (music on console)를 설치하려면:

[source, shell]
....
# pkg install moc
....

[[vlc]]
=== VLC media player

VLC media player는 다양한 오디오 및 비디오 형식(MPEG-1, MPEG-2, MPEG-4, DivX, mp3, ogg 등)은 물론 DVD, VCD 및 다양한 스트리밍 프로토콜을 위한 휴대성이 뛰어난 멀티미디어 플레이어입니다. 또한 고대역폭 네트워크에서 IPv4 또는 IPv6의 유니캐스트 또는 멀티캐스트 스트리밍을 위한 서버로 사용할 수도 있습니다. 또한 VLC는 스트리밍 또는 디스크 저장을 위해 미디어를 즉석에서 트랜스코딩할 수 있는 기능도 제공합니다.

VLC를 설치하려면:

[source, shell]
....
# pkg install vlc
....

Kodi를 설치하려면:

[source, shell]
....
# pkg install kodi
....

[[conferencing-meetings]]
== 회의 및 미팅

FreeBSD 데스크탑 환경을 사용하여 화상 회의에 참여할 수 있습니다. 이 섹션에서는 웹캠을 구성하는 방법과 FreeBSD에서 지원되는 화상 회의 애플리케이션에 대해 설명합니다.

[[webcam-setup]]
=== 웹캠 설정하기

FreeBSD가 웹캠에 액세스하고 구성을 수행하도록 허용하려면 특정 유틸리티를 설치해야 합니다:

* package:multimedia/webcamd[]은 수백 가지의 다양한 USB 기반 웹캠 및 DVB USB 장치를 사용할 수 있는 데몬입니다.
* package:multimedia/pwcview[]는 웹캠의 비디오 스트림을 보는 데 사용할 수 있는 애플리케이션입니다.

필요한 유틸리티를 설치하려면:

[source, shell]
....
# pkg install webcamd pwcview
....

`/etc/rc.conf` 에서 man:webcamd[8] 서비스를 활성화하여 시스템 부팅 시 시작하도록 합니다:

[source, shell]
....
# sysrc webcamd_enable="YES"
....

사용자는 `webcamd` 그룹에 속해야 합니다. 사용자를 `webcamd` 그룹에 추가하려면 다음 명령을 실행합니다:

[source, shell]
....
# pw groupmod webcamd -m username
....

package:multimedia/webcamd[]에는 man:cuse[3] 모듈이 필요하므로 다음 명령을 실행하여 이 모듈을 로드해야 합니다:

[source, shell]
....
# kldload cuse
....

시스템 부팅 시 man:cuse[3]를 로드하려면 다음 명령을 실행합니다:

[source, shell]
....
# sysrc kld_list += "cuse"
....

유틸리티가 설치되면 사용 가능한 웹캠 목록은 man:webcamd[8]로 확인할 수 있습니다:

[source, shell]
....
# webcamd -l
....

출력은 다음과 비슷해야 합니다:

[.programlisting]
....
webcamd [-d ugen0.2] -N SunplusIT-Inc-HP-TrueVision-HD-Camera -S unknown -M 0 <.>
webcamd [-d ugen1.3] -N Realtek-802-11n-WLAN-Adapter -S 00e04c000001 -M 0
....
<.> 사용 가능한 웹캠

다음 명령을 실행하여 사용 가능한 웹캠을 구성합니다:

[source, shell]
....
# sysrc webcamd_0_flags="-d ugen0.2" <.>
....

[NOTE]
====
Plug-and-play USB 웹캠인 경우 연결된 USB 포트를 변경하면 `webcamd -l` 의 출력이 변경되므로 rc.conf의 항목을 업데이트해야 할 수도 있습니다. USB 통합 웹캠을 사용하는 노트북의 경우 문제가 되지 않습니다.
====

다음 명령을 실행하여 man:webcamd[8] 서비스를 시작합니다:

[source, shell]
....
# service webcamd start
....

출력은 다음과 비슷해야 합니다:

[.programlisting]
....
Starting webcamd.
webcamd 1616 - - Attached to ugen0.2[0]
....

package:multimedia/pwcview[]를 사용하여 웹캠이 제대로 작동하는지 확인할 수 있습니다. package:multimedia/pwcview[]를 실행하려면 다음 명령을 사용합니다:

[source, shell]
....
% pwcview -f 30 -s vga
....

그러면 package:multimedia/pwcview[]가 웹캠을 표시합니다:

[[scanners]]
== 이미지 스캐너

FreeBSD에서 이미지 스캐너에 대한 액세스는 link:http://www.sane-project.org[SANE (Scanner Access Now Easy)]를 통해 제공되며, 이 링크는 FreeBSD 포트 컬렉션에서 사용할 수 있습니다.

[[scanners-kernel-usb]]
=== 스캐너 확인하기

구성을 시도하기 전에 스캐너가 SANE에서 지원되는지 확인하는 것이 중요합니다.

스캐너가 연결된 상태에서 다음 명령을 실행하여 연결된 모든 USB 장치를 가져옵니다:

[source, shell]
....
# usbconfig list
....

출력은 다음과 비슷해야 합니다:

[.programlisting]
....
ugen4.2: <LITE-ON Technology USB NetVista Full Width Keyboard.> at usbus4, cfg=0 md=HOST spd=LOW (1.5Mbps) pwr=ON (70mA)
ugen4.3: <Logitech USB Optical Mouse> at usbus4, cfg=0 md=HOST spd=LOW (1.5Mbps) pwr=ON (100mA)
ugen3.2: <HP Deskjet 1050 J410 series> at usbus3, cfg=0 md=HOST spd=HIGH (480Mbps) pwr=ON (2mA)
....

다음 명령을 실행하여 `idVendor` 및 `idProduct` 를 가져옵니다:

[source, shell]
....
# usbconfig -d 3.2 dump_device_desc
....

[NOTE]
====
스캐너는 plug-and-play 장치이므로 연결된 USB 포트를 변경하면 `usbconfig list` 의 출력이 변경된다는 점에 유의하세요.
====

출력은 다음과 비슷해야 합니다:

[.programlisting]
....
ugen3.2: <HP Deskjet 1050 J410 series> at usbus3, cfg=0 md=HOST spd=HIGH (480Mbps) pwr=ON (2mA)

bLength = 0x0012
bDescriptorType = 0x0001
bcdUSB = 0x0200
bDeviceClass = 0x0000  <Probed by interface class>
bDeviceSubClass = 0x0000
bDeviceProtocol = 0x0000
bMaxPacketSize0 = 0x0040
idVendor = 0x03f0
idProduct = 0x8911
bcdDevice = 0x0100
iManufacturer = 0x0001  <HP>
iProduct = 0x0002  <Deskjet 1050 J410 series>
bNumConfigurations = 0x0001
....

`idVendor` 와 `idProduct` 를 획득한 후, 해당 스캐너가 `idProduct` 로 필터링하여 지원되는지 link:http://www.sane-project.org/lists/sane-mfgs-cvs.html[SANE의 지원 장치 목록]에서 확인해야 합니다.

[[_sane_configuration]]
=== SANE 구성

SANE은 백엔드를 통해 스캐너에 대한 액세스를 제공합니다. FreeBSD로 스캔하려면 다음 명령을 실행하여 package:graphics/sane-backends[] 패키지를 설치해야 합니다:

[source, shell]
....
# pkg install sane-backends
....

[TIP]
====
일부 USB 스캐너는 펌웨어를 로드해야 합니다. 위의 예에서 사용된 HP 스캐너처럼 package:print/hplip[] 패키지가 설치되어 있어야 합니다.
====

필요한 패키지를 설치한 후 스캐너에 대한 FreeBSD 액세스를 허용하도록 man:devd[8]를 구성해야 합니다.

`saned.conf` 파일을 다음 내용과 함께 [.filename]#/usr/local/etc/devd/saned.conf# 에 추가합니다:

[.programlisting]
....
notify 100 {
        match "system" "USB";
        match "subsystem" "INTERFACE";
        match "type" "ATTACH";
        match "cdev" "ugen[0-9].[0-9]";
        match "vendor" "0x03f0"; <.>
        match "product" "0x8911"; <.>
        action "chown -L cups:saned /dev/\$cdev && chmod -L 660 /dev/\$cdev";
};
....

<.> `vendor` : 이전에 `usbconfig -d 3.2 dump_device_desc` 명령을 실행하여 얻은 idVendor입니다.
<.> `product` : 이전에 `usbconfig -d 3.2 dump_device_desc` 명령을 실행하여 얻은 idProduct입니다.

그 후 다음 명령을 실행하여 man:devd[8]를 재시작해야 합니다:

[source, shell]
....
# service devd restart
....

SANE 백엔드에는 장치를 나열하고 이미지 수집을 수행하는 데 사용할 수 있는 man:scanimage[1]가 포함되어 있습니다.

man:scanimage[1]을 `-L` 인수와 함께 실행하여 스캐너 장치를 나열합니다:

[source, shell]
....
# scanimage -L
....

출력은 다음과 비슷해야 합니다:

[.programlisting]
....
device `hpaio:/usb/Deskjet_1050_J410_series?serial=XXXXXXXXXXXXXX` is a Hewlett-Packard Deskjet_1050_J410_series all-in-one
....

man:scanimage[1]가 스캐너를 식별할 수 없는 경우 이 메시지가 표시됩니다:

[.programlisting]
....
No scanners were identified. If you were expecting something different,
check that the scanner is plugged in, turned on and detected by the
sane-find-scanner tool (if appropriate). Please read the documentation
which came with this software (README, FAQ, manpages).
....

man:scanimage[1]가 스캐너를 확인하면 구성이 완료되고 이제 스캐너를 사용할 준비가 된 것입니다.

서비스를 활성화하고 부팅 시 실행되도록 하려면 다음 명령을 실행합니다:

[source, shell]
....
# sysrc saned_enable="YES"
....

man:scanimage[1]를 사용하여 명령줄에서 이미지 캡처를 수행할 수 있지만, 이미지 스캔을 수행할 때는 그래픽 인터페이스를 사용하는 것이 더 좋습니다.
